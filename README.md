# 計画書進捗ミニボード

PT 個人用の、1ページ完結型「計画書進捗ミニボード」です。  
スプレッドシートより少ない操作で進捗を更新できるように、患者ごとの工程更新を大きなトグルボタン中心で設計しています。

## 特徴

- React + TypeScript + Vite で構築
- データ保存は `localStorage` のみ
- 患者をカードUIで一覧表示
- `文書作成` `サイン済` `提出・コスト算定済` をワンタップで即更新
- 3工程がすべて完了するとカード色が変わり、一覧下部へ自動移動
- 上部に未完了数、完了数、各待ち件数を集計表示
- `未完了のみ表示` の切り替え対応
- CSV エクスポート / インポート対応
- 初回起動時はサンプルデータを表示
- スマホでも使いやすいレスポンシブUI

## 起動方法

Node.js 18 以上を用意してから、以下を実行してください。

```bash
npm install
npm run dev
```

ブラウザで、表示されたローカル URL を開くと利用できます。

## ビルド方法

```bash
npm run build
npm run preview
```

## 使い方

1. 画面左の `患者追加` フォームで患者名とリハ開始日を登録します。
2. 追加された患者カード上で、3つの大きい工程ボタンを押して進捗を更新します。
3. 完了条件を満たすと、自動で完了色に変わり一覧下部へ移動します。
4. `未完了のみ表示` をオンにすると、進行中の患者だけを素早く確認できます。
5. `CSVエクスポート` でバックアップ保存、`CSVインポート` で復元や置き換えができます。

## データ仕様

各患者データは次の項目を保持します。

- `id`
- `patientName`
- `rehabStartDate`
- `documentCreated`
- `signed`
- `submitted`
- `memo`
- `updatedAt`

保存先はブラウザの `localStorage` です。キーは `pt-plan-progress-mini-board` です。

## 主要コンポーネント

### `src/App.tsx`

画面全体のレイアウト、CSV 入出力、通知メッセージ制御を担当します。

### `src/hooks/usePatientBoard.ts`

患者データの状態管理をまとめたカスタムフックです。  
`localStorage` 保存、並び替え、進捗更新、削除、CSV インポート反映を一元化しています。

### `src/components/AddPatientForm.tsx`

最小入力で患者を追加するフォームです。  
登録後すぐにカード一覧へ反映されます。

### `src/components/SummaryBar.tsx`

未完了件数、完了件数、各工程待ち件数を上部で一覧表示します。

### `src/components/Toolbar.tsx`

未完了のみ表示の切り替え、CSV エクスポート / インポート、サンプル再読み込みを担当します。

### `src/components/PatientBoard.tsx`

患者カード一覧の描画を担当します。

### `src/components/PatientCard.tsx`

患者ごとの進捗更新UIです。  
工程ボタン、完了バッジ、メモ、最終更新日時、削除操作をまとめています。

### `src/utils/patientUtils.ts`

完了判定、並び替え、日付表示、CSV 変換などの純粋関数を集約しています。
